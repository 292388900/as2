Schlüssel und Zertifikate

Signatur und Verschlüsselung von Nachrichten

Verschlüsselung des Übertragungsweges (https)

 

Das AS2 Protokoll erlaubt eine verschlüsselte Verbindung von Daten über das Internet. Zudem lassen sich die Nachrichten signieren und ebenfalls verschlüsseln. Daher ist es wichtig, entsprechende Schlüssel vorzuhalten.

Dazu ist eine Schlüsselverwaltung in die AS2 Server GUI integriert. Die Terminologie der in der GUI bezeichnet einen "Schlüssel" als den private Schlüssel oder ein Schlüsselpaar (privater und öffentlicher Schlüssel), während "Zertifikat" ein Zertifikat meint, in das ein öffentlicher Schlüssel eines Partner eingebettet ist.

 

Signatur und Verschlüsselung von Nachrichten

Der Sender einer Nachricht benötigt jeweils einen privaten Schlüssel für sich und einen öffentlichen Schlüssel seines Partners.
Eine Nachricht wird mit dem jeweiligen privaten Schlüssel signiert, dieser sollte sich daher nur im Besitz des jeweiligen Senders befinden. Der öffentliche Schlüssel dient zur Verifikation einer Signatur. Der gleiche Mechanismus funktioniert auch beim Verschlüsseln.

Die öffentlichen Schlüssel sind vor der Übertragung entsprechend auszutauschen. Um dem Partner seinen öffentlichen Schlüssel zu geben, kann die Funktion "Zertifikat exportieren" in der GUI zum Verwalten der Zertifikate benutzt werden. Um den öffentlichen Schlüssel (eingebettet ins Zertifikat) seines Partners zu importieren, kann die Funktion "Schlüssel-Zertifikat importieren" benutzt werden.
Jeder Schlüssel oder jedes Schlüsselpaar wird über einen Alias adressiert, der beim Import zu vergeben ist. Bei den Partnerdefinitionen kann dann jeweils dieser Alias ausgewählt werden.
Bitte beachten Sie, dass die meisten Schlüssel eine begrenzte Gültigkeit haben, das sollten Sie von Zeit zu Zeit überprüfen. Das Ablaufdatum ist in der GUI zur Schlüsselverwaltung ersichtlich. Wenn ein Schlüssel abgelaufen ist, kann mit ihm nicht mehr gearbeitet werden, er ist ungültig.
Ein Nachrichtenaustausch kann nur signiert und/oder verschlüsselt stattfinden, wenn beide beteiligten Parteien über die jeweils gleichen Zertifikate und Schlüssel verfügen.

 

Verschlüsselung des Übertragungsweges (https)

Der Http Übertragungsweg wird gesondert verschlüsselt (ssl). Dazu sind die entsprechenden Zertifikate zum Senden von Nachrichten in den Keystore zu importieren, der in den Einstellungen des Servers angegeben ist. Ein Import von Zertifikaten in diesen Keystore kann über das java Kommandozeilentool "keytool" stattfinden. Sehr empfehlenswert ist jedoch ein Programm, das direkt mit Keystores umgehen kann, wie zum Beispiel "Portecle" (Open Source).
Der mendelson AS2 Server wird in der Regel mit einem Tomcat HTTP Server ausgeliefert. Dieser ist schon so konfiguriert, dass unter tomcat/common der Keystore tomcat.keystore mit dem Passwort "changeit" liegt, in den die entsprechenden Clientzertifikate der Partner importiert werden können.

Falls Sie diesen vorkonfigurierten Server nicht benutzen wollen, schauen Sie bitte in die Dokumentation Ihres jeweiligen Servers, wie dort benutzerdefinierte Clientzertifikate für Https einzubinden sind.
